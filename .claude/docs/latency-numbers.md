# Latency Numbers Every Programmer Should Know

*(Jeff Dean / Peter Norvig, ~2012)*

## CPU・メモリ系（ns オーダー）

| 操作 | レイテンシ | 備考 |
|------|-----------|------|
| L1 キャッシュ参照 | **0.5 ns** | 最速（数クロックで実行可能） |
| 分岐予測ミス | **5 ns** | 数十クロックのロス |
| L2 キャッシュ参照 | **7 ns** | L1の14倍 |
| Mutex lock/unlock | **25 ns** | — |
| メインメモリ参照 | **100 ns** | L2の20倍、L1の200倍 |

## I/O・ネットワーク系（μs〜ms オーダー）

| 操作 | レイテンシ | 備考 |
|------|-----------|------|
| 1KB を Zippy 圧縮 | **3 μs** (3,000 ns) | — |
| 1Gbps NW で 1KB 送信 | **10 μs** (10,000 ns) | — |
| SSD からランダム 4KB 読み出し | **150 μs** (150,000 ns) | ~1GB/s SSD |
| メモリから 1MB シーケンシャル読み出し | **250 μs** (250,000 ns) | — |
| 同一DC内ラウンドトリップ | **500 μs** (500,000 ns) | — |
| SSD から 1MB シーケンシャル読み出し | **1 ms** (1,000,000 ns) | メモリの4倍 |
| HDD シーク | **10 ms** (10,000,000 ns) | DC往復の20倍 |
| HDD から 1MB シーケンシャル読み出し | **20 ms** (20,000,000 ns) | メモリの80倍、SSDの20倍 |
| CA → オランダ → CA パケット往復 | **150 ms** (150,000,000 ns) | 大陸間通信 |

## 比率の感覚

```
L1(0.5ns) → L2(7ns) → RAM(100ns) → SSD random(150μs) → HDD seek(10ms)
          14倍       14倍          1500倍               67倍

同一DC(500μs) << 大陸間(150ms) = 300倍の差
```

## 設計への示唆

- **メモリ vs SSD random**: 約1500倍 → インメモリキャッシュの価値が大きい
- **SSD vs HDD**: シーク不要なSSDはHDDシーク込みの約67倍速い
- **同一DC vs 大陸間**: 300倍の差 → レイテンシクリティカルな処理は地理的に近いノードで完結させる
- **ネットワーク 1KB送信(10μs) < メモリ 1MB読み出し(250μs)**: データサイズが重要
